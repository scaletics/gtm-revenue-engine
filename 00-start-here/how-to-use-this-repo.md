## Purpose

This guide explains how to use the GTM Revenue OS as an internal operating system and as a reusable client delivery template.

The repository is designed for fast-moving teams that need repeatable GTM execution without creating disconnected decks, docs, and one-off playbooks.

---

## Core Operating Model

Use this repo in three layers:

1. **Core GTM system**  
   Universal process, positioning, discovery, proposal, and sales execution standards.

2. **Segment overlays**  
   Adjustments for SMB SaaS, mid-market SaaS, and MSP buyers.

3. **Client customization layer**  
   Client-safe templates adapted for specific engagements.

---

## First-Time Setup

### Step 1: Confirm the Use Case

Decide whether the work is for:

- Internal Scaletics execution
- Client GTM engagement
- Sales opportunity support
- Partner or channel development
- Proposal or SOW creation
- Market or competitor research

### Step 2: Select the Segment

Use one of the following segments:

| Segment | When to Use |
|---|---|
| SMB SaaS | Founder-led sales, simple buying motion, short sales cycle |
| Mid-Market SaaS | Multiple stakeholders, business case, procurement, 3 to 6 month cycle |
| MSP | Recurring managed services, IT, cloud, cybersecurity, Microsoft, infrastructure |

### Step 3: Start with the Correct Folder

| Task | Start Here |

## How Sales Should Use This Repo

Before a first call:

1. Confirm the segment.
2. Review the relevant segment overlay.
3. Use the qualification checklist.
4. Prepare discovery questions.
5. Review relevant objections.

After discovery:

1. Capture current state.
2. Capture desired future state.
3. Identify business impact.
4. Map the decision process.
5. Select the right proposal or follow-up asset.

Before proposal:

1. Confirm pain.
2. Confirm success criteria.
3. Confirm decision stakeholders.
4. Confirm investment range.
5. Confirm next meeting.

---

## How BD Should Use This Repo

For partnerships:

1. Review `04-business-development/partnership-strategy.md`.
2. Score partner fit using the partner qualification scorecard.
3. Select the right motion: referral, channel, co-marketing, alliance, marketplace.
4. Build the partner value proposition.
5. Use QBR and alliance templates to manage the relationship.

---

## How Client Delivery Should Use This Repo

For client work:

1. Complete client intake.
2. Select the segment overlay.
3. Copy only the client-safe templates.
4. Replace internal language.
5. Remove internal pricing, private examples, and Scaletics-only operating notes.
6. Deliver as a clean client-specific GTM operating system.

---

## What Not to Do

Do not:

- Create duplicate sales processes by segment.
- Put client-private information in shared templates.
- Edit pricing or positioning without owner review.
- Add templates that are not ready for real use.
- Use vague file names like `final-v2-new.md`.
- Let outdated docs remain active.
